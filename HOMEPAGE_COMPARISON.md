# Порівняння: HomePage (CODIA_main_inzur) vs головна сторінка (sitesucker index.html)

## Джерела

| Проєкт | Шлях | Технологія |
|--------|------|------------|
| **Наш** | `CODIA_main_inzur/src/pages/HomePage.tsx` | React + Vite + Tailwind |
| **Референс** | `sitesucker_inzhur/www.inzhur.reit/index.html` | Nuxt (SSR), один великий HTML + `_nuxt/*.css` |

---

## 1. Структура сторінки

| Секція | Sitesucker (оригінал) | Наш HomePage |
|--------|------------------------|--------------|
| **Hero / Intro** | Є: `intro.CH6q5RVX.css`, `Intro.BuuUugnA.css` — заголовок, мапа, плашка «Поріг входу», блок статистики (intro-facts) | Є: той самий порядок — h1, плашка, статистика (3 блоки з count-up) |
| **Пропозиції** | Є (chapter-home) | Є: «Пропозиції Inzhur» — 3 картки (REIT, Енергетика, ОВДП) |
| **Головне про інвестиції** | Є (howItWorks, text-block) | Є: «Що таке Inzhur?» + «Як це працює?» |
| **Подарункові сертифікати** | Є: `giftCertificates.*.css` | Є: градієнтний блок + «Обрати сертифікат» |
| **Відео та питання** | Є: `videoFAQ`, `videoBox` | Є: placeholder відео + FAQ посилання |
| **Документи** | Є: `docs.*` | Є: «Документи та прозорість» |
| **Мапа об'єктів** | Є: на мобільному показується заголовок «Мапа об'єктів» (`map-title` display:none на desktop, на max-width:767px — display:inline) | **Немає** окремої секції «Мапа об'єктів» під блоком «Поріг входу» на вузьких екранах |

**Висновок:** У нас є всі основні блоки. Відмінність — у оригіналі на мобільному є заголовок «Мапа об'єктів» і, ймовірно, сама мапа нижче; у нас цієї секції немає.

## 2. Hero (перший блок)

| Елемент | Sitesucker (з intro.CH6q5RVX.css) | Наш HomePage |
|---------|-----------------------------------|--------------|
| **Заголовок** | `.section-title`: 44px (1700px: 66px), letter-spacing -2.64px, #101820 | `text-[32px] hero:text-[40px]`, tracking -1.5px / -2.64px, #10171f — **близько** |
| **Плашка «Поріг входу»** | `.intro-description`: bg #134169, border-radius 27px (1700px: 32px), max-width 335px (1700px: 400px), padding 24px (1700px: 28px) | `bg-[#134169] rounded-[32px]`, `hero:max-w-[400px]`, `p-8 hero:p-10` — **відповідає** |
| **Фон (мапа)** | `.map-wrapper` + `.map-img`: на 768–1199px інше позиціонування; на &lt;767px мапа 360px висоти | У нас: при ≥1201px — `1st_block_backgound.webp`; при &lt;1201 — hero-bg.png або без мапи (залежно від логіки) |
| **Брейкпоінти** | 767px, 768–1199px, 1200px, 1700px | У нас: `hero: 764px`, `header: 1201px` — **трохи інші**, але аналогічно |

**Висновок:** Логіка та стилі Hero узгоджені з оригіналом; ми додали явне розділення desktop/mobile для плашки (≥1201 — в Hero, &lt;1201 — окремий блок знизу).

## 5. Технічні відмінності

| Аспект | Sitesucker (index.html) | CODIA_main_inzur (HomePage) |
|--------|--------------------------|-----------------------------|
| **Розмітка** | Один великий HTML (Nuxt SSR), контент і дані в `__NUXT_DATA__` | React-компонент, структура в JSX |
| **Стилі** | Багато окремих CSS-файлів у `_nuxt/` (intro, Intro, entry, giftCertificates, videoFAQ, docs тощо) | Tailwind-класи в компоненті + `index.css` (шрифти, `.hero-first-block-bg`) |
| **Брейкпоінти** | 767px, 768px, 1199px, 1200px, 1700px у різних CSS | У нас: 764px (`hero`), 1201px (`header`) в `tailwind.config.js` |
| **Фони** | Мапа через `.map-wrapper` / `.map-img`; окремі картинки в _nuxt | У нас: `1st_block_backgound.webp` (≥1201px), hero-bg.png (&lt;1201px), потім прибрано hero-bg на desktop |
| **Логотип** | SVG/лого в header | У нас: символ «꩜» у заокруглених квадратах |
| **Хедер** | `header--sticky`, `.content-grid` | У нас: окремий `Header` / `DashboardHeader`, sticky при скролі, на початку — колір як у сторінки |

## 6. Що є в оригіналі, чого немає у нас

1. **Секція «Мапа об'єктів»** — на оригіналі на мобільному є заголовок «Мапа об'єктів» і, ймовірно, блок з мапою під плашкою «Поріг входу». У нас цієї секції немає.
2. **Точні брейкпоінти** — оригінал використовує 767/768, 1199/1200, 1700; ми — 764 та 1201 (навмисно для узгодженості з header/burger).
3. **Оригінальні зображення/іконки** — ми використовуємо частину асетів з sitesucker (наприклад hero-bg), але лого замінено на «꩜», мапа — на `1st_block_backgound.webp`.

---

## 7. Що у нас реалізовано додатково або інакше

1. **Адаптивна плашка «Поріг входу»**: при &lt;1201px окремий блок знизу з закругленими лише верхніми кутами; при ≥1201px — картка в Hero.
2. **Фон-мапа тільки на desktop**: при ≥1201px показується `1st_block_backgound.webp`; старий hero-bg на desktop приховується.
3. **Анімовані цифри** в блоці статистики (GSAP + ScrollTrigger).
4. **Єдиний контейнер** контенту: `max-w-[1280px]`, `px-4 md:px-8 lg:px-[5vw]` для всіх секцій.
5. **Доступність**: `aria-hidden` для декоративних елементів, `aria-live="polite"` для цифр, skip-link у Layout.
